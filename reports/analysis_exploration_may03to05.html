    Exploring the Order ID Format in the Dataset

    As part of my exploratory data analysis (EDA), I focused on understanding the various patterns (formats) of Order IDs in the dataset. Here's a breakdown of my findings:

    1. Extracted Digit Patterns and Recorded Hyphen-Delimited Part Lengths:

  • The Order ID format consists of a pattern with three segments: 3 digits - 7 digits - 7 digits.
  • Total Order IDs: 128,975
  • Malformed Order IDs: 0

    • 2. Unique Record Count by Order ID:

    I analyzed the dataset to identify unique records for each Order ID.

  • Total records: 128,975
  • Unique Order IDs: 120,378
  • Missing Order IDs: 0
  • Valid records (duplicates excluded): 120,378
  • Duplicate Order IDs: 8,597
  • Order IDs not following the pattern "3 digits - 7 digits - 7 digits": 124

    • 3. Duplicated Order IDs: 8,597 and Non-Pattern Order IDs: 124:

    I further investigated the 124 records that did not follow the '3 digits - 7 digits - 7 digits' pattern, and found that all of them followed a slightly different pattern: '1 letter + 2 digits - 7 digits - 7 digits.' Separately, I explored the 8,597 duplicated Order IDs, which all followed the '3 digits - 7 digits - 7 digits' pattern.

  • Duplicated Order IDs: 8,597 (exported to CSV)
  • Malformed Order IDs (following a different pattern): 124
